Configurando ELK Windows

Pré-requisitos

  1. Windows 11 instalado.
  1. Java 8 ou superior (necessário para Logstash).
  1. RAM: Mínimo 4 GB (ideal 8 GB+ para o ELK Stack rodar bem).
  1. Permissões de administrador.
  1. Elastic Stack (Elasticsearch, Logstash e Kibana).

Passo 1: Instalação do Java (pré-requisito para Logstash)

  1. Faça o download do Java JDK:
    • Acesse: Download OpenJDK ou Oracle JDK.
    • Baixe o instalador correspondente ao seu sistema (Windows x64).
  1. Instale o Java JDK:
    • Siga as instruções no instalador.
  1. Configure a variável de ambiente JAVA_HOME:
    • Clique com o botão direito no Menu Iniciar → Sistema → Configurações avançadas do sistema.
    • Em Variáveis de Ambiente, adicione:
      • Nome: JAVA_HOME
      • Valor: C:\Program Files\Java\jdk-<sua-versão>.
  1. Teste a instalação:
    Abra o
    Prompt de Comando (cmd) e digite:
    
java -version

Passo 2: Instalação do Elasticsearch

  1. Baixe o Elasticsearch:
    • Acesse: Elastic Downloads.
    • Baixe o ZIP da versão mais recente.
  1. Extraia o arquivo em uma pasta, por exemplo:

    C:\elk\elasticsearch

  1. Configure o arquivo elasticsearch.yml:
    • Acesse C:\elk\elasticsearch\config\elasticsearch.yml com um editor de texto (como VSCode ou Notepad++).
    • Adicione ou modifique as seguintes linhas:
      
network.host: 127.0.0.1
http.port: 9200
  1. Inicie o Elasticsearch:
    • No Prompt de Comando, vá até a pasta bin:
      
cd C:\elk\elasticsearch\bin
elasticsearch.bat

-- Aqui confira o caminho, na minha maquina o caminho era cd C:\elk\elasticsearch\elasticsearch-8.17.0-windows-x86_64\elasticsearch-8.17.0\bin
    • Aguarde a inicialização (leva alguns minutos).
    • Teste no navegador:Acesse http://localhost:9200.Deverá aparecer um JSON com as informações do Elasticsearch.
    • Obs: Se der erro tente https://localhost:9200, vai ser necessário uma senha, então faça:

    1. Abra o Prompt de Comando (cmd) ou PowerShell.
    1. Navegue até a pasta bin do Elasticsearch:
      
cd C:\elk\elasticsearch\bin

-- Lembre de mudar seu C: para o caminho da sua maquina
    1. Execute o seguinte comando para redefinir a senha do usuário elastic:
      
elasticsearch-reset-password -u elastic
    1. Resultado esperado:
      O comando vai gerar uma nova senha. A saída será algo assim:
      
Password for the [elastic] user successfully reset.
New value: 4h8dT#xk2oB9QhYl

Passo 3: Instalação do Kibana

  1. Baixe o Kibana:
    • Acesse: Elastic Downloads.
    • Baixe o ZIP correspondente.
  1. Extraia o conteúdo para uma pasta:

    C:\elk\kibana.

  1. Configure o arquivo kibana.yml:
    • Acesse C:\elk\kibana\config\kibana.yml e modifique:
      
server.port: 5601
server.host: "127.0.0.1"
elasticsearch.hosts: ["http://localhost:9200"]
  1. Inicie o Kibana:
    • No Prompt de Comando, vá para a pasta bin:
      
cd C:\elk\kibana\bin
kibana.bat

-- Lembre de mudar seu C: para o caminho da sua maquina
    • Aguarde a inicialização.
    • Acesse o Kibana no navegador: http://localhost:5601.

Passo 4: Instalação do Logstash

  1. Baixe o Logstash:
    • Acesse: Elastic Downloads.
    • Baixe o ZIP do Logstash.
  1. Extraia o conteúdo para uma pasta:

    C:\elk\logstash.

  1. Configure o arquivo logstash.conf:
    • Crie um arquivo chamado logstash.conf em C:\elk\logstash\config.
    • Adicione a configuração básica para ingerir logs simples:
      
input {
  file {
    path => "C:/elk/logs/teste.log"
    start_position => "beginning"
  }
}

output {
  elasticsearch {
    hosts => ["http://localhost:9200"]
    index => "logstash-test"
  }
  stdout { codec => rubydebug }
}

-- Lembre de mudar seu C: para o caminho da sua maquina
  1. Crie um arquivo de logs para teste:
    • No diretório C:\elk\logs, crie um arquivo chamado teste.log e adicione algumas linhas:
      
[INFO] Log de teste - Primeira linha
[ERROR] Log de teste - Erro fictício
  1. Execute o Logstash:
    • No Prompt de Comando, vá até a pasta bin do Logstash:
      
cd C:\elk\logstash\bin
logstash.bat -f C:\elk\logstash\config\logstash.conf
  1. Verifique no Elasticsearch:
    • No navegador, acesse:http://localhost:9200/logstash-test/_search.
    • Os logs deverão aparecer no JSON.

Passo 5: Visualização de Dados no Kibana

  1. Acesse o Kibana:

    http://localhost:5601.

  1. Configure o Index Pattern:
    • No menu lateral, vá em Management → Stack Management → Index Patterns.
    • Clique em Create Index Pattern.
    • Nome do padrão: logstash-*.
    • Configure o campo de tempo, se necessário.
  1. Explore os dados:
    • Vá até Discover no menu lateral.
    • Seus logs deverão aparecer listados.
  1. Crie um Dashboard:
    • Vá em Dashboard → Create New Dashboard.
    • Adicione gráficos, tabelas ou visualizações baseados nos dados de logs.

Resumo das Portas Utilizadas

  • Elasticsearch: Porta 9200
  • Kibana: Porta 5601
  • Logstash: Conexões configuradas nos arquivos.

Demonstração Completa

  1. Elasticsearch está rodando em http://localhost:9200.
  1. Kibana está rodando em http://localhost:5601 e mostra os logs.
  1. Logstash está ingerindo o arquivo teste.log e enviando os dados para o Elasticsearch.
